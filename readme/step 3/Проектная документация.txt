Проектная документация для API, отвечающего за создание заявки на выпуск пропуска гостю, получение списка активных пропусков гостей и блокировку пропуска гостя

Цель проекта
Разработать API для создания заявки на выпуск пропуска гостю, получение списка активных пропусков гостей и блокировки пропуска гостя.

Целевая аудитория
API предназначен для использования в системе управления доступом на территорию организации, для работы с пропусками для гостей.

Требования к API

API должен быть реализован с использованием протокола HTTP и формата данных JSON.
API должен поддерживать аутентификацию и авторизацию пользователей.
API должен обеспечивать безопасность передаваемых данных и защиту от несанкционированного доступа.
API должен обеспечивать возможность создания заявки на выпуск пропуска гостю, получение списка активных пропусков гостей и блокировки пропуска гостя.

Описание запросов API

1. Создание заявки на выпуск пропуска гостю
URL: /DoRequestGuest
Метод: GET

Параметры запроса:

user_id (строка, обязательный) - логин из личного кабинета
inn (строка, обязательный) - ИНН компании
id_remote (строка, обязательный) - Уникальный для заявки, он же номер заявки

FLastName (строка, обязательный) - фамилия гостя
FFirstName (строка, обязательный) - имя гостя
FMiddleName (строка, необязательный) - отчество гостя
FCarNumber (строка, необязательный) - гос. номер автомобиля
FDateFrom (строка, обязательный) - дата (без времени) начала визита в формате "yyyy-mm-dd"
FDateTo (строка, обязательный) - дата (без времени) окончания визита в формате "yyyy-mm-dd"

Ответ:

RESULT (строка) - варианты где SUCCESS успешное выполнение запроса и ERROR возникла ошибка при выполнении запроса
DESC (строка) - описание ошибки
DATA () - без данных

2. Получение списка активных заявок для гостей на компанию

URL: /GetGuestsStatus
Метод: GET
Параметры запроса:

user_id (строка, обязательный) - логин из личного кабинета
inn (строка, обязательный) - ИНН компании

Ответ:

RESULT (строка) - варианты где SUCCESS успешное выполнение запроса и ERROR возникла ошибка при выполнении запроса
DESC (строка) - описание ошибки
DATA (список) - содержит информацию по всем активным пропускам для гостей
пример данных гостя:
    DateFrom_Request (строка) - дата начала визита в формате "yyyy-mm-dd"
    DateTo_Request (строка) - дата окончания визита в формате "yyyy-mm-dd"
    Date_Request (строка) - дата и время создания заявки "yyyy-mm-dd HH:MM:SS"
    FStatus (строка) - статус пропуска
    Name_FirstName (строка) - имя гостя
    Name_LastName (строка) - фамилия гостя
    Name_MIddleName (строка) - отчество гостя
    Number_Car (строка) - номер автомобиля

3. Блокировка пропуска гостя

URL: /DoBlockGuest
Метод: GET
Параметры запроса:

user_id (строка, обязательный) - логин из личного кабинета
inn (строка, обязательный) - ИНН компании

Ответ:

RESULT (строка) - варианты где SUCCESS успешное выполнение запроса и ERROR возникла ошибка при выполнении запроса
DESC (строка) - описание ошибки
DATA () - без данных

Безопасность
Аутентификация и авторизация
API должен поддерживать аутентификацию и авторизацию пользователей.
Для этого в каждом запросе должны быть поля "user_id" и "inn". Где "user_id" это логин в личном кабинете и "inn" это ИНН компании
Так же используется механизм доступа по IP.
