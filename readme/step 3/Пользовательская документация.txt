Проектная документация для API, которая отвечает за создание заявки на выпуск пропуска гостю,
получения списка пропусков для гостей и блокировки пропуска гостя.

Описание API
Аутентификация и авторизация происходит в запросе через поля "user_id" и "inn".

Описание запросов API

1. Запрос на получение списка активных заявок гостей на компанию

    Вид запроса
    Метод: GET
    URL: /GetGuestsStatus

    Параметры запроса:
    Пример запроса: 127.0.0.1:8077/GetGuestsStatus

    Принимает json данные.
    {
        "user_id": "логин из личного кабинета",
        "inn": "инн компании"
    }

Ответ:
    Отвечает json
    {
        "RESULT": "SUCCESS\ERROR",
        "DESC": "описание при ошибке",
        "DATA": "список активных заявок гостей на компанию"
    }

2. Запрос на создание заявки на выпуск пропуска гостю

    Вид запроса
    Метод: GET
    URL:  /DoRequestGuest

    Параметры запроса
    Пример запроса: 127.0.0.1:8077/DoRequestGuest

    Принимает json данные.
    {
        "user_id": "логин из личного кабинета",
        "inn": "инн компании",
        "id_remote": "id" (Уникальный для заявки, он же номер заявки)

        "FLastName": "text",
        "FFirstName": "text",
        "FMiddleName": "text",
        "FCarNumber": "text",
        "FDateFrom": "дата по дням",
        "FDateTo": "дата по дням",
    }

Ответ:

    Отвечает json
    {
        "RESULT": "SUCCESS\ERROR",
        "DESC": "описание при ошибке",
        "DATA": ""
    }

3. Запрос на блокировку пропуска гостя

    Вид запроса
    Метод: GET
    URL: /DoBlockGuest

    Параметры запроса
    Пример запроса:
    127.0.0.1:8077/DoBlockGuest

    Принимает json данные.
    {
        "user_id": "логин из личного кабинета",
        "inn": "инн компании",
        "id_remote": "id" (Уникальный id заявки из ЛК)
    }

Ответ:

    Отвечает json
    {
        "RESULT": "SUCCESS\ERROR\WARNING",
        "DESC": "описание при ошибке",
        "DATA": ""
    }
